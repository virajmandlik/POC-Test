## 1. Project Overview

### What Is This?

A backend pipeline system that:

- Ingests documents (PDF/Image) via `/ingress` API
- Processes them through quality checks, OCR extraction (offline + online), and intelligent merging
- Outputs structured JSON via `/egress` API

### Who Uses It?

- **Farmers** — upload documents, verify extracted personal/land data
- **Government Officials** — verify, approve, and consume structured data
- **Farmers for Forests (F4F)** — downstream consumer of structured land records for carbon credit programs

### Core Requirements

| Requirement          | Detail                                                  |
|----------------------|---------------------------------------------------------|
| Languages Supported  | Marathi, Hindi, English                                 |
| Offline Capability   | Full pipeline must work without internet                |
| Online Enhancement   | When network available, use GPT Vision + GPT-4o-mini    |
| Output Format        | Structured JSON                                         |
| Accuracy Target      | >90% for printed text, >75% for handwritten             |
| Lightweight          | Offline model should be <500MB                          |
| Primary Document     | Maharashtra 7/12 Land Record (gaav namuna saat)         |

### Extraction Routing Logic

| Scenario     | What Happens                                                                          |
|--------------|---------------------------------------------------------------------------------------|
| Online       | PaddleOCR + GPT Vision run in parallel --> Both outputs merged via GPT-4o-mini        |
| Offline      | Only PaddleOCR runs --> Output formatted directly into JSON                           |
| GPT Timeout  | GPT Vision times out --> Fall back to PaddleOCR-only result (no pipeline block)       |

**Merge Prompt Template (implemented in combined.py):**

```
SYSTEM:
You are a document data extraction merger. Your job is to combine
two independent extractions of the same document into one accurate result.

EXTRACTION A (Offline OCR - PaddleOCR):
{offline_ocr_output_json}

EXTRACTION B (Online Vision - GPT Vision API):
{gpt_vision_output_json}

RULES:
1. Compare both extractions field by field.
2. Where both agree -> use that value, confidence = HIGH.
3. Where they disagree -> prefer higher confidence source.
4. Where one is missing -> use available value.
5. Do NOT hallucinate data not present in either extraction.

OUTPUT FORMAT:
Return ONLY valid JSON matching this schema:
{output_json_schema}
```

## 6. Component Specifications

### 6.1 Preprocessing Module

```
File: src/preprocessing/image_processor.py

class ImageProcessor:
    Methods:
        convert_pdf_to_images(pdf_path, dpi=300) -> List[ImagePath]
        deskew(image_path) -> ImagePath
        denoise(image_path) -> ImagePath
        binarize(image_path) -> ImagePath
        enhance_contrast(image_path) -> ImagePath
        check_quality(image_path) -> QualityReport
        preprocess_pipeline(input_path) -> PreprocessResult

    Quality Thresholds (configurable via config.yaml):
        blur_threshold: 100.0
        min_resolution: 150
        brightness_range: [50, 220]
        overall_pass_threshold: 0.6
```

### 6.2 Extraction Module

```
File: src/extraction/engine.py

class ExtractionEngine:
    Methods:
        extract_offline(image_path, language) -> ExtractionResult
        extract_online(image_path, language) -> ExtractionResult
        extract_dual(image_path, language) -> MergedExtractionResult
        merge_results(offline_result, online_result) -> MergedExtractionResult

    Routing Logic:
        IF network_available AND offline_confidence < 0.80:
            -> dual extraction (parallel)
        ELIF network_available AND offline_confidence >= 0.80:
            -> offline only (save cost)
        ELSE:
            -> offline only (no choice)

    Fallback Chain:
        1. Merged result (best)
        2. GPT-only result
        3. Offline-only result
        4. FAIL
```

### 6.3 Formatting Module

```
File: src/formatting/formatter.py

class DocumentFormatter:
    Methods:
        format(extraction_result, schema) -> FormattedDocument
        validate(formatted_doc) -> ValidationResult
        map_fields(raw_text, field_rules) -> Dict[str, FieldValue]

    Field Mapping Rules (configurable via YAML):
        - name: regex patterns for name detection
        - date: multiple date format parsers
        - aadhaar: 12-digit pattern with checksum
        - phone: 10-digit Indian mobile pattern
        - address: multi-line grouping logic
        - pincode: 6-digit pattern

    Missing Field Handling:
        - Mandatory field missing -> flag as INCOMPLETE
        - Optional field missing -> set null
        - Low confidence field -> include but flag
```

## 11. Risk Matrix

| Risk                                        | Probability | Impact | Severity    | Mitigation                                                                              |
|---------------------------------------------|-------------|--------|-------------|-----------------------------------------------------------------------------------------|
| Low OCR accuracy on handwritten Indic text   | High        | High   | CRITICAL    | Invest in preprocessing; fine-tune model Phase 3; flag low-confidence for human review  |
| GPT API downtime / rate limiting             | Medium      | Medium | MODERATE    | Offline fallback always available; retry with backoff; never block pipeline on API       |
| High GPT API costs at scale                  | Medium      | Medium | MODERATE    | Confidence-based routing; cost tracking; budget alerts                                  |
| PII data sent to OpenAI servers              | High        | High   | CRITICAL    | PII masking/redaction before API call; review data policies; consider Azure OpenAI       |
| Network unavailability in rural areas        | High        | Medium | MODERATE    | Full offline pipeline is core requirement; sync when connected                          |
| Verification bottleneck (slow human review)  | Medium      | Medium | MODERATE    | Auto-approve high-confidence; batch verification; simple UI for speed                   |
| Mixed language documents causing confusion   | Medium      | Medium | MODERATE    | Script detection + language routing; test extensively on mixed docs                     |
| Model drift over time                        | Low         | Medium | LOW         | Periodic retraining with correction data; model version tracking                        |
| Multi-page PDF processing failures           | Low         | Low    | LOW         | Process pages independently; if one page fails continue others                          |

## 13. Quick Start

```bash
# Clone repository
git clone <repo-url>
cd docextract-pipeline

# Create virtual environment
python -m venv venv
# Linux/Mac: source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Download OCR models
bash scripts/download_models.sh

uvicorn src.main:app --reload --host 0.0.0.0 --port 8000

curl -X POST http://localhost:8000/ingress \

  -F "language_hint=marathi"
```
